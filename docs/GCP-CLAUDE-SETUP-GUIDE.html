1. What is already built in your repo

You do not need to write the Python/HTML generator from scratch. The codebase already includes:

ComponentLocationRole
DB → portfolio.mdsrc/spec_builder.pyReads Supabase; no AI inventing user facts
Claude Code buildsrc/build_engine.pyNext.js template + optional Claude CLI + npm build
Flask APIsrc/main.pyPOST /build for n8n (300s timeout)
Next templatetemplates/portfolio-next/Static export; loads data.json

n8n workflow v2infra/n8n/workflows/bexo-portfolio-generate.jsonNo OpenRouter AI nodes

Your job in GCP: deploy this container, wire secrets, connect n8n + Cloudflare. The code is ready; infrastructure is what you configure below.

2. How it works

BEXO App (profile ≥90%) → api-server POST /portfolio/trigger-build → n8n webhook (Workflow A) → Cloud Run POST /build [Claude Code + DeepSeek/Kimi proxy] 1. spec_builder.py pulls Supabase 2. Next.js build → out/ 3. Upload to gs://bexo-sites-public/{profileId}/site/ → n8n callback (Workflow B) → Cloudflare KV + Supabase site_builds = done → https://handle.mybexo.com (Worker → GCS) User edits project in app → POST /api/portfolio/sync-data → Overwrites gs://.../site/data.json only (cheap, no Cloud Run)

Claude vs DeepSeek (important)

The container runs the Claude Code CLI, but API calls are routed through ANTHROPIC_BASE_URL to DeepSeek (or OpenRouter). You need a DeepSeek API key, not necessarily an Anthropic subscription.

3. Prerequisites

  • Google Cloud project (e.g. bexo-prod) with billing enabled
  • gcloud CLI installed and logged in: gcloud auth login
  • DeepSeek API key — platform.deepseek.com
  • Optional: Kimi (Moonshot) key for fix pass — platform.moonshot.ai
  • Supabase project URL + service role key (already used by BEXO)
  • n8n running (e.g. n8n.mybexo.com)
  • Cloudflare zone for mybexo.com
  • GitHub PAT for optional portfolio-{handle} repos

Phase C — Secret Manager (AI + platform)

Create secrets (skip if they already exist; use versions add to update):

# Required for Claude Code proxy (DeepSeek)
echo -n "YOUR_DEEPSEEK_KEY" | gcloud secrets create deepseek-api-key --data-file=- 2>/dev/null \
  || echo -n "YOUR_DEEPSEEK_KEY" | gcloud secrets versions add deepseek-api-key --data-file=-

# Optional: Kimi fix pass
echo -n "YOUR_KIMI_KEY" | gcloud secrets create kimi-api-key --data-file=- 2>/dev/null \
  || echo -n "YOUR_KIMI_KEY" | gcloud secrets versions add kimi-api-key --data-file=-

# Optional: OpenRouter fallback
echo -n "sk-or-..." | gcloud secrets create openrouter-api-key --data-file=- 2>/dev/null \
  || echo -n "sk-or-..." | gcloud secrets versions add openrouter-api-key --data-file=-

# Platform (likely exist already)
echo -n "$(openssl rand -hex 32)" | gcloud secrets create bexo-internal-secret --data-file=-
echo -n "https://YOUR.supabase.co" | gcloud secrets create supabase-url --data-file=-
echo -n "eyJ..." | gcloud secrets create supabase-service-key --data-file=-
echo -n "github_pat_..." | gcloud secrets create github-token --data-file=-
echo -n "https://n8n.mybexo.com/webhook/bexo-build-done" | gcloud secrets create n8n-callback-url --data-file=-

GitHub org (required for portfolio repos): Cloud Run uses GITHUB_ORG=bexo-sites (not BEXO-DEV). PAT must have repo + workflow scopes; token owner must be a member of bexo-sites. Verify before deploy:

export GITHUB_TOKEN="$(gcloud secrets versions access latest --secret=github-token --project=$PROJECT_ID)"
./scripts/verify-github-phase5.sh bexo-sites

Grant bexo-codegen@ access to every secret:

SA="bexo-codegen@${PROJECT_ID}.iam.gserviceaccount.com"
for SECRET in deepseek-api-key kimi-api-key openrouter-api-key github-token \
  supabase-url supabase-service-key bexo-internal-secret n8n-callback-url; do
  gcloud secrets add-iam-policy-binding "$SECRET" \
    --member="serviceAccount:${SA}" \
    --role="roles/secretmanager.secretAccessor" --quiet 2>/dev/null || true
done

Phase D — Deploy Cloud Run (Claude Code engine)

D.1 Build and deploy image

cd bexo-codegen
gcloud builds submit --config=cloudbuild.yaml --project=$PROJECT_ID

This builds a Docker image with Node 20, Python, Chromium, Playwright, and @anthropic-ai/claude-code. Deploy sets 4Gi RAM and 300s timeout.

D.2 Add Claude proxy environment variables (critical)

cloudbuild.yaml maps DEEPSEEK_API_KEY but you must also set the Anthropic-compatible proxy URL and model names on the Cloud Run service:

gcloud run services update bexo-codegen \
  --region=us-central1 \
  --project=$PROJECT_ID \
  --update-env-vars="ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic,ANTHROPIC_MODEL=deepseek-chat,ANTHROPIC_DEFAULT_SONNET_MODEL=deepseek-chat,CLAUDE_CODE_SUBAGENT_MODEL=deepseek-chat,CODEGEN_ENGINE=claude,SKIP_CLAUDE=false"

If DeepSeek’s Anthropic bridge URL changes, update ANTHROPIC_BASE_URL per their docs. For template-only test (no Claude CLI billing): set SKIP_CLAUDE=true.

D.3 Confirm service URL

gcloud run services describe bexo-codegen --region=us-central1 --format='value(status.url)'

Save this URL — you need it for n8n as CLOUD_RUN_BUILD_URL (append /build only in the workflow body; base URL without path).

D.4 Smoke test /build

INTERNAL=$(gcloud secrets versions access latest --secret=bexo-internal-secret)
URL=$(gcloud run services describe bexo-codegen --region=us-central1 --format='value(status.url)')

curl -X POST "${URL}/build" \
  -H "Content-Type: application/json" \
  -H "X-BEXO-Internal-Secret: ${INTERNAL}" \
  -d '{"profileId":"YOUR_PROFILE_UUID","buildId":"YOUR_BUILD_UUID","handle":"yourhandle"}'

First run can take 3–8 minutes (npm install + build). Check logs for GitHub: https://github.com/bexo-sites/portfolio-HANDLE (not GitHub skipped):

gcloud run services logs read bexo-codegen --region=us-central1 --limit=80

D.5 Verify GCS output

gsutil ls -l "gs://bexo-sites-public/YOUR_PROFILE_UUID/site/"
gsutil ls "gs://bexo-sites-public/YOUR_PROFILE_UUID/site/_next/"
gsutil cat "gs://bexo-sites-public/YOUR_PROFILE_UUID/site/data.json" | head -20

Rollback

If Claude path fails, switch to legacy HTML generator:

gcloud run services update bexo-codegen --region=us-central1 \
  --update-env-vars="CODEGEN_ENGINE=python"

Phase E — n8n Workflow A

Re-import the updated workflow (no OpenRouter AI nodes):

bexo-codegen/infra/n8n/workflows/bexo-portfolio-generate.json

On your n8n VM, edit ~/bexo-n8n/.env:

N8N_WEBHOOK_SECRET=<same as BEXO api-server>
BEXO_INTERNAL_SECRET=<same as GCP bexo-internal-secret>
SUPABASE_URL=https://xxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJ...
CLOUD_RUN_BUILD_URL=https://bexo-codegen-XXXX.us-central1.run.app
N8N_BLOCK_ENV_ACCESS_IN_NODE=false

Restart n8n:

cd ~/bexo-n8n && docker compose -f docker-compose.micro.yml up -d --force-recreate n8n

In n8n UI: import workflow → publish. Confirm the Cloud Run — /build node timeout is 300000 ms (5 minutes).

You can remove OPENROUTER_API_KEY from n8n for Workflow A — spec is built inside Cloud Run now.

Phase F — Cloudflare (live URLs)

Full details: docs/PORTFOLIO-HOSTING-PLAN.md

  1. Run setup-public-sites-bucket.sh if not done
  2. Create KV namespace → note CLOUDFLARE_KV_NAMESPACE_ID
  3. Deploy worker from infra/cloudflare/worker/ (wrangler)
  4. Wildcard DNS *.mybexo.com → Worker route
  5. Workflow B writes handle → profileId to KV on build success
cd bexo-codegen/infra/cloudflare/worker
npx wrangler deploy

Phase G — api-server (sync edits without rebuild)

When a student edits a project and the site is already live, the app calls POST /api/portfolio/sync-data to refresh data.json only.

On your api-server host / BEXO/.env:

GCS_PUBLIC_BUCKET=bexo-sites-public
# Paste full service account JSON (GCS write to bexo-sites-public):
GCS_SA_KEY_JSON={"type":"service_account",...}

Create a GCP service account with roles/storage.objectAdmin on bexo-sites-public, download JSON key, set as env var. Run pnpm install in api-server (adds @google-cloud/storage).

Phase H — Test end-to-end

  1. Student profile <90% → app shows banner; build returns 403
  2. Complete profile → tap Build My Website on dashboard
  3. site_builds.status → building → done
  4. Open https://handle.mybexo.com — Next.js site loads
  5. Edit a project title in app → wait ~3s → refresh site → title updated
  6. Optional: Redesign in app runs full Cloud Run build again

See also docs/PHASE-8-APP-TEST-CHECKLIST.md and docs/IMPLEMENTATION_VERIFICATION.md.
